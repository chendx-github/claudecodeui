# 修改账号密码脚本使用说明（中文版）

适用项目：`/home/chendx/claudecodeui`  
脚本路径：`scripts/change-user-password.mjs`

## 1. 功能说明

这个脚本用于修改登录账号的密码（写入 `auth.db` 的 `users.password_hash`），支持三种输入方式：

- 明文参数：`--password`（方便但不推荐，可能进入历史记录）
- 环境变量：`--password-env`（推荐自动化）
- 交互输入：`--prompt`（推荐手动操作，输入不回显）

## 2. 常用命令

先进入项目目录：

```bash
cd /home/chendx/claudecodeui
```

### 2.1 交互输入（推荐）

```bash
node scripts/change-user-password.mjs --username chendx --prompt
```

说明：
- 会要求输入两次密码（默认二次确认）
- 输入内容不会显示在终端

如果不想二次确认：

```bash
node scripts/change-user-password.mjs --username chendx --prompt --no-confirm
```

### 2.2 环境变量方式（推荐脚本化）

```bash
PASSWORD_VALUE='ChendxUI123.' node scripts/change-user-password.mjs --username chendx --password-env PASSWORD_VALUE
```

### 2.3 明文参数（不推荐）

```bash
node scripts/change-user-password.mjs --username chendx --password 'ChendxUI123.'
```

## 3. 高级参数

- 指定数据库路径：

```bash
node scripts/change-user-password.mjs --username chendx --prompt --db /path/to/auth.db
```

- 指定 bcrypt 轮数（默认 `12`）：

```bash
node scripts/change-user-password.mjs --username chendx --prompt --rounds 12
```

## 4. 快速排查

- `User not found`：用户名不对，可先检查账号名是否为 `chendx`
- `Database file not found`：数据库路径不对，检查 `--db` 或 `DATABASE_PATH`
- `Passwords do not match`：两次输入不一致，重新执行命令

## 5. 安全建议

- 优先使用 `--prompt` 或 `--password-env`
- 不建议直接用 `--password`（容易泄露到命令历史）
- 修改成功后建议退出当前登录并重新登录验证
