# ClaudeCodeUI 启动与服务使用说明（中文版）

本文档说明两件事：
- 如何手动启动（前台运行）
- 如何注册为服务（后台运行）

## 1. 前置条件

- 操作系统：Linux（其他系统命令类似）
- Node.js：建议 `v22+`
- npm：可正常使用
- 项目目录：`/home/chendx/claudecodeui`

首次使用建议先执行：

```bash
cd /home/chendx/claudecodeui
npm install
cp .env.example .env
```

按需修改 `.env` 中端口：
- `PORT=3001`（后端/API）
- `VITE_PORT=5173`（开发模式前端端口）

## 2. 手动启动（前台）

### 2.1 开发模式（热更新）

```bash
cd /home/chendx/claudecodeui
npm run dev
```

访问：
- `http://localhost:3001`

说明：
- 该模式适合改代码调试。
- 停止：在当前终端按 `Ctrl + C`。

### 2.2 生产模式（先构建再启动）

```bash
cd /home/chendx/claudecodeui
npm run build
npm run server
```

访问：
- `http://localhost:3001`

说明：
- 该模式适合稳定运行。
- 停止：在当前终端按 `Ctrl + C`。

## 3. 注册为服务（推荐 PM2）

PM2 适合 Node 应用常驻后台运行，并支持开机自启。

### 3.1 安装并启动服务

```bash
npm i -g pm2
pm2 start npm --name claudecodeui --cwd /home/chendx/claudecodeui -- run server
pm2 save
pm2 status
```

### 3.2 常用管理命令

```bash
pm2 restart claudecodeui
pm2 stop claudecodeui
pm2 start claudecodeui
pm2 logs claudecodeui
pm2 delete claudecodeui
```

### 3.3 设置开机自启

```bash
pm2 startup
```

执行命令后会打印一条需要 `sudo` 执行的命令，复制执行它，然后再运行：

```bash
pm2 save
```

## 4. 可选方案：systemd（原生服务）

如果你不想用 PM2，可以使用 `systemd`。以下是最小示例。

### 4.1 创建服务文件

文件路径：`/etc/systemd/system/claudecodeui.service`

内容示例：

```ini
[Unit]
Description=ClaudeCodeUI Service
After=network.target

[Service]
Type=simple
User=chendx
WorkingDirectory=/home/chendx/claudecodeui
ExecStart=/usr/bin/npm run server
Restart=always
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

### 4.2 启用并启动

```bash
sudo systemctl daemon-reload
sudo systemctl enable claudecodeui
sudo systemctl start claudecodeui
sudo systemctl status claudecodeui
```

查看日志：

```bash
journalctl -u claudecodeui -f
```

## 5. 故障排查

- 端口占用：
  - 检查 `PORT` 是否冲突，必要时改 `.env` 后重启。
- 页面打不开：
  - 检查服务进程是否存在（`pm2 status` 或 `systemctl status`）。
  - 检查防火墙是否放行对应端口。
- 安卓无法访问：
  - 使用局域网地址访问：`http://电脑局域网IP:3001`
  - 手机与电脑需在同一局域网。
